# draw.io Agent Drawing Skill

Use this skill when an Agent needs to create, update, or manage reviewable draw.io / diagrams.net diagrams.

## Planning

1. Identify the diagram type, audience, key nodes, relationships, and review criteria.
2. Use Mermaid for flowcharts, architecture flows, sequence-style drafts, and state diagrams when the layout can be generated by diagrams.net.
3. Use draw.io XML when exact layout, shape libraries, annotations, or freeform whiteboarding matter.
4. Read the current diagram before updating an existing user-edited diagram.
5. Preserve the user's latest XML unless the request is a full replacement.

## Mermaid Path

- Save Mermaid with `drawio_save_mermaid_draft`.
- The Workbench loads Mermaid through the diagrams.net JSON protocol descriptor `{ format: "mermaid", data }`.
- The user reviews the imported diagram in the embedded editor and saves the resulting XML as a new version.
- Keep Mermaid concise and syntactically valid.

## XML Path

- Save complete diagrams.net XML with `drawio_create_diagram` or `drawio_save_scene_version`.
- Prefer an `<mxfile>` wrapper with at least one `<diagram>` page.
- Use stable labels and simple shape semantics so future edits remain understandable.
- For small updates, call `drawio_get_diagram` first and then `drawio_patch_scene` with replacement XML or related metadata.

## Versioning

- Every meaningful Agent or Workbench change should become a version with a short `changeSummary`.
- `draft` means still being prepared or reviewed.
- `reviewed` means a human accepted the current version.
- `archived` means it is removed from the active workbench.

## Failure Reporting

Call `drawio_report_failure` when source material cannot be converted, XML is invalid, or a requested change would destroy user edits.
