# 演示文稿工作室

在 Xpert 中与 AI 协作者一起创建精美、可编辑、可演示、可导出的专业演示文稿。

演示文稿工作室可以把报告、项目简报、会议记录、结构化数据或内容提纲，转化为一份能够持续审阅和修改的完整演示文稿。它把理解演示逻辑的智能体、原生可视化 Workbench、多人实时协作、显式版本管理和生产级导出组合在同一个 Agentic App 中。

## 从想法到完整演示稿

你可以让演示文稿工作室根据资料和目标直接制作演示文稿。演示文稿助手可以：

- 为整份文稿选择统一主题，并为每一页匹配合适版式；
- 把原始资料整理成清晰的汇报叙事；
- 创建、排序和修改幻灯片；
- 根据汇报目标、受众、汇报人和页数要求调整内容；
- 关联 Xpert 工作空间中已有的图片和媒体；
- 检查文案长度、必填字段、媒体引用和版式约束；
- 导出 HTML、PDF 和 PPTX。

打开已有文稿后，你也可以直接要求助手“修改当前文稿”或“调整当前页”。Workbench 会把正在打开的文稿、当前幻灯片和编辑上下文提供给智能体，使后续请求准确作用于你正在查看的内容，而不是重新创建一份无关文稿。

## 真正可编辑的工作室

演示文稿工作室不是一次性生成图片式 PPT 的工具。全屏 Workbench 提供：

- 幻灯片导航、选择和组织；
- 高保真中央编辑画布；
- 文本原位编辑和可移动编辑元素；
- 根据当前版式动态生成的设计检查器；
- 当前页面支持的颜色、数值、开关、选项和媒体控件；
- 本地编辑撤销和重做；
- 版本、导出、素材和校验面板；
- 播放、缩放和适应画布操作。

界面会继承 Xpert 宿主主题变量，各个面板独立滚动，并且只加载当前文稿所需的主题 Runtime。

## 14 套主题，1,190 个版式

插件完整包含 DashiAI 的 14 套主题和 1,190 个演示文稿版式，新增“亮色循环商业”（`theme13`）和“橄榄建筑提案”（`theme14`），覆盖：

- 封面、章节页和过渡页；
- 目录与章节导航；
- 关键指标、数字卡片和对比分析；
- 时间线、路线图和流程图；
- 产品、团队、战略和项目总结；
- 图片、视频、图表和混合媒体页面；
- 结论、行动建议和结束页。

一份文稿只使用一套统一主题。智能体会先按页面角色搜索版式，再检查版式支持的字段和控件后填入内容，减少文案溢出、布局破坏和不支持属性等问题。

## 根据你的外部模板生成主题

在 Workbench 中选择**文稿 → 根据外部模板生成主题**，输入主题名称，明确选择来源契约，然后上传 React 项目/压缩包、HTML 页面/压缩包、PPTX、PDF、包含 8–30 页图片的 ZIP 或混合压缩包。上传只会把源文件保存到 Workspace Files 并创建当前作用域的**已准备**主题，不会启动后台任务；随后 Presentation Studio 中间件会直接打开插件内置的 `dashi-theme-generator` 流程和作者工程，不需要 `skillsMiddleware`，也不需要另装 Skill。在对话中，智能体也可以接收 8–30 个显式图片文件引用，不会再把多个路径拼成一个字符串。

生成器会依次进入**分析中 → 生成中 → 验证中**，盘点源文件、选择显式适配器、提取 Style DNA，并构建真正可编辑的模块，不会把截图平铺成幻灯片背景。注册要求 76–96 个版式、至少 8 个有外部证据的 observed 模块、8 个基于 Style DNA 的 inferred 模块、9 个结构家族，并通过能力、配色、自有模块渲染、画布/控件、运行时契约和全页库排版门槛。任一门槛失败都会保留具体错误并禁用主题；只有从“验证中”成功注册的主题才可像内置主题一样用于 Workbench 和 HTML/PDF/PPTX 导出。

## 多人与多智能体共同编辑

多个用户可以实时编辑同一份工作文稿。Presentation Studio 会展示在线协作者、实时光标、文本选区、当前页面，以及其他协作者正在操作的元素或设计控件。

智能体也使用同一套协作机制。当智能体添加页面、修改文本、调整控件或处理媒体元素时，它会作为 AI 协作者出现在协作者列表中，并展示当前操作和目标位置。人与智能体的更新进入同一份权威协作文档，不会通过整页覆盖互相冲掉修改。

协作者在线状态是临时信息，不会写入演示文稿内容或版本历史。

## 工作副本与版本

日常编辑会持续更新当前工作副本。输入文字、多人协作、播放和导出都不会频繁创建不可变版本。

只有用户点击**保存版本**，或者明确要求智能体保存或定稿版本时，系统才会创建新版本。之后可以查看历史版本，并把旧版本恢复为一个新的工作节点，而不会覆盖原始历史。

这样既能保持实时协作顺畅，也能让每个重要评审节点清晰可追踪。

## 导出与分享

根据交付场景选择输出格式：

| 格式 | 适合场景 |
| --- | --- |
| **HTML** | 自包含浏览器演示、在线评审和外部链接分享。 |
| **PDF** | 稳定评审稿、讲义和文档交付。 |
| **PPTX** | PowerPoint 兼容交付，尽可能保留可编辑文字、形状和图片。 |

复杂视觉效果在 PPTX 中可能采用截图降级，并在上层恢复可编辑文字。导出报告会说明字体替代、内容溢出、不支持元素和截图降级等情况。

HTML 导出可以通过 Xpert Artifacts 发布。公开分享必须由用户显式触发；分享出去的只有演示文稿产物，不会暴露对话内容、工作空间、协作 session 或平台凭证。

生产环境中的 PDF 和 PPTX 会进入 Xpert 的 Sandbox 浏览器任务池，并在短生命周期 Job Sandbox 中执行。Xpert 统一 Browser Runtime 提供 Playwright、Chromium、字体和通用 Runner Host；插件只发布版本化的 `presentation.export` Sandbox Action Bundle。Xpert 自动注册 `browser/playwright-1.61/v1`。PDF/PPTX 能力探测默认开启；Action、Runtime artifact、Provider 或 Worker 不可用时显示可操作的 warning，HTML 仍可独立使用。

OSS 基础部署有意不启动 Sandbox Runtime Worker，也不挂载 Docker socket。兼容 Provider 发行物可以提供自己的 Worker 部署；Xpert Pro 会随 Docker Provider 打包 Worker overlay 和不可变 Runtime artifact lock，终端用户无需配置 Provider、Profile、Image、`CHROME_PATH` 或功能开关。

`exportBackend: local` 与 `chromiumExecutablePath`/`CHROME_PATH` 仅作为已废弃的开发、测试兼容配置保留。生产环境始终强制使用 `exportBackend: sandbox-job`，不会在 API 容器内启动或下载 Chromium。

## 典型使用流程

1. 可选：上传外部模板，让智能体生成并注册一套可复用主题。
2. 提供目标、受众、汇报人、来源材料和期望页数。
3. 让助手建立文稿结构并选择每页版式。
4. 在 Workbench 中审阅并修改文字、设计属性、媒体和页面顺序。
5. 邀请同事协作，或让智能体继续优化当前页面。
6. 在重要评审节点保存版本。
7. 导出 HTML、PDF 或 PPTX。
8. 需要对外分享时，发布 HTML Artifact 链接。

## 适合场景

- 项目方案与管理层汇报；
- 年度、季度和月度经营总结；
- 产品发布和战略规划；
- 研究分析与数据故事；
- 培训材料和内部赋能；
- 销售、咨询和客户演示；
- 希望使用 AI 提效，同时保留人工直接编辑和审阅能力的团队。

## 内容与隐私

- 文稿、版本、导出和素材均限定在当前 Xpert 租户和组织作用域内。
- 媒体通过 Workspace Files 引用，不会把大文件正文放进智能体工具结果。
- 协作 session 有效期短，并且只允许一个用户访问一份文稿。
- 系统不会静默创建公开 Artifact 链接。
- 用户文本只会写入版式已知字段，不会被当作任意 HTML 注入。

## 技术来源与许可证

Presentation Studio 保留 DashiAI PPT Skill 基线 commit `69ac66443e36e11cfca4a7f30721dc71a4278d28`，并同步已安装 Dashi PPT 0.4.0 中 `theme13`/`theme14` 的源码、证据、元数据和 Runtime，形成 14 套主题、1,190 个版式的完整目录。插件同时打包 `dashi-theme-generator` 方法和作者工程，用于生成可移植、经验证的自定义主题包。上游源码、完整性信息、版权归属和第三方声明均保留在插件中。

演示字体不会再以二进制文件形式提交到上游 vendor 源码目录。插件将所需的 OFL-1.1 Fontsource 包作为精确版本的运行时依赖，渲染时识别当前主题使用的字体族，只为该主题生成字体包；HTML 导出再把 WOFF2 数据和生成的许可证清单内联到自包含文件中。这样既减小 Git 历史与插件源码包，也不会增加 CDN 请求或破坏离线导出能力。
