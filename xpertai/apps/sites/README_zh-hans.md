# 网站发布工作室

无需离开 Xpert，就能把一个需求转化为精美、可审阅、可发布的网站。

Sites 是用于创建、审阅、版本化、发布和分享静态网站的 Agentic App。你只需描述想要的网站，让 Sites Assistant 在 Xpert 工作空间中生成文件，在 Workbench 中检查结果，然后通过 Xpert Artifacts 发布受控部署。

## 可以创建什么

Sites 适合制作自包含的网页体验，例如：

- 落地页和营销活动页面；
- 团队门户和员工入职中心；
- 培训赋能和资源中心；
- 产品、项目和发布专题页；
- 活动规划中心与日程页面；
- 创意收集和优先级看板；
- 轻量数据看板和状态页面；
- 作品集、报告和交互式说明页面。

Sites Assistant 可以使用用户提示词、对话中的文件、笔记和链接开始制作。如果没有提供真实来源材料，模板会使用清晰可修改的示例内容，不会把虚构数据描述成从公司系统中发现的真实信息。

## 让智能体制作，由用户掌控发布

你可以用自然语言要求 Sites Assistant 创建或修改网站。它可以：

- 创建 Sites 项目并组织源文件；
- 生成静态 HTML、CSS 和 JavaScript；
- 检查项目元数据和已有版本；
- 保存可审阅的不可变版本；
- 发布已批准版本；
- 更新访问权限；
- 发布或撤销 Artifact 链接；
- 查询部署状态并报告可操作的失败信息。

Workbench 会集中展示项目、版本、部署、访问权限、环境值和预览。你可以先审阅一个版本，再决定是否发布，而不是让智能体的每次修改都立即替换线上内容。

## 从成熟模板开始

Sites 提供多种常用网站起点，包括：

- 员工入职中心；
- 销售或团队赋能中心；
- 企业运营 Pulse Dashboard；
- 活动规划中心；
- 创意收集工作区；
- 产品发布日历。

模板只是起点，不是不可修改的主题。你可以让智能体调整信息架构、视觉方向、内容、交互和目标受众，同时保持结果为可部署的静态网站。

## 版本与部署

Sites 将编辑、保存和发布明确分开：

1. **项目**用于标识网站及其源文件位置。
2. **版本**是一次不可变、可审阅的网站快照。
3. **部署**使用指定访问策略发布一个已批准版本。
4. **Artifact Link**是由平台管理的打开和分享入口。

创建或修改文件不会静默覆盖已发布网站。先保存新版本，检查确认后再部署。

根据产品流程，Artifact 链接可以跟随最新部署内容，也可以保持指向一个受控的发布结果。链接标识、鉴权、有效期、撤销和访问审计统一由平台管理。

## 访问与分享

可以选择谁有权访问部署：

| 访问范围 | 适合场景 |
| --- | --- |
| **仅管理员** | 私有管理和早期评审。 |
| **工作空间成员** | 当前工作空间中的内部网站。 |
| **自定义受众** | 限定的用户或用户组评审。 |
| **公开链接** | 互联网上任何获得链接的人。 |

公开发布始终需要用户显式确认。Sites 不自行拼接公开 URL、不保存公开 token，也不暴露插件专用的公开 Hosting Controller；最终链接由 Xpert Artifacts 创建和鉴权。

撤销链接会立即关闭对应公开访问入口，但不会删除内部项目和已经保存的版本。

## 典型使用流程

1. 描述网站目的、目标受众、内容和期望视觉方向。
2. 让助手在 `/workspace/sites/<slug>/` 中创建网站源文件。
3. 在 Sites Workbench 中审阅项目，并继续提出修改要求。
4. 内容达到评审状态后保存不可变版本。
5. 使用合适的访问模式部署已批准版本。
6. 复制 Artifact 链接用于内部或外部分享。
7. 后续更新保存为新版本，并保留旧版本用于追踪。

## 工作空间内容与环境值

网站源文件保存在 Xpert 工作空间中。部署产物通过 Workspace Files 保存，并由不可变 Artifact Version 引用。

Workbench 可以管理项目环境值元数据，并对标记为 secret 的值进行遮罩。静态发布页面不能嵌入平台凭证或私密环境值；环境值记录用于部署配置管理，不应用来向浏览器 JavaScript 暴露密钥。

## 适合场景

- 希望直接从智能体对话发布实用网站的团队；
- 需要工作空间权限或自定义访问范围的内部门户；
- 可以完全在浏览器端运行的轻量交互工具；
- 需要版本审阅和可追踪发布流程的场景；
- 需要可撤销、可审计公开链接的对外页面；
- 不想单独搭建前端项目和 Hosting Pipeline 的用户。

## 当前产品边界

Sites 当前发布静态、自包含的网站内容：根目录 `index.html`，以及可选 CSS、JavaScript 和打包资源。

- 当前部署模型不包含服务端路由、数据库、Worker 或任意后端 Runtime。
- 浏览器交互可以按需使用客户端 JavaScript 和 `localStorage`。
- 只有在 Assistant 已安装并获得用户授权时，才会使用相应连接器。
- 当前版本不提供自定义域名和多路由动态应用托管。
- 公开部署只分享生成的网站，不会公开原始对话和工作空间。

## Xpert 平台能力

Sites 使用 Workspace Files 保存可移植部署内容，使用 Xpert Artifacts 管理不可变版本、安全预览、受控链接、有效期、撤销和访问日志。这样既能让网站发布流程与智能体和 Workbench 保持连接，又不会让生成页面获得平台凭证。
