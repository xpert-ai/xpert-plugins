# Output Templates

Use these templates as content guides for PM-quality event output. For an explicit full event analysis, full report, or reusable/source-heavy special-situations package, default to a polished standalone HTML event report following `../../../shared/html-artifact-standard.md` and the event-specific HTML guidance in the owning skill. Do not force a fixed section inventory when evidence or event type calls for a different structure. Use the quick read only when the user explicitly asks for a quick, short, summary, one-page, brief, or TL;DR output.

## HTML Full Event Report

Use flexible standalone HTML for a substantive full event report. Lead with a decision posture that matches the evidence stage and put the event type's primary analytical object near the top. If live pricing or execution evidence required to act is missing, label the artifact as an underwriting framework or entry screen, state what is needed to initiate, and avoid an executable trade headline.

For a spin-off or split-off, prioritize distribution mechanics and hard dates, the value-allocation bridge, SpinCo and RemainCo starting points, confirmed technical-flow evidence versus thesis, entry screen, monitoring, and red team. For merger arbitrage, show spread, downside, implied probability, and scenario math only when supportable. For regulatory or litigation events, prioritize procedural path, remedy risk, and payoff implications.

Use the full content inventory below as applicable. Replace unsupported market-pricing, probability, scenario, or expected-return modules with evidence gaps, valuation screens, and gating inputs rather than presenting false precision.
